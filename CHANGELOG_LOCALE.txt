================================================================================
FIXES & IMPROVEMENTS
================================================================================

------------------------------------------------------------------------------
1. FLOATING POINT STABILITY (classifyRemainder)
------------------------------------------------------------------------------

ISSUE:
- CONVENTIONAL_FRACTIONS.includes(remainder) was unreliable
- Floating point errors: 0.1 + 0.2 = 0.30000000000000004
- Triplets (1/3, 2/3) were not handled correctly

SOLUTION:
- Added EPSILON = 0.001 tolerance constant
- Added isApproximatelyEqual(a, b) helper function
- Added TRIPLET_FRACTIONS = [1/3, 2/3] array
- Replaced .includes() with tolerance-based comparison

CODE ADDED:
```typescript
const EPSILON = 0.001;
const TRIPLET_FRACTIONS = [1/3, 2/3];

function isApproximatelyEqual(a: number, b: number): boolean {
  return Math.abs(a - b) < EPSILON;
}
```

------------------------------------------------------------------------------
2. DEBUG LOGGING FOR BPM GROWTH
------------------------------------------------------------------------------

ISSUE:
- No visibility into when BPM growth triggered
- Difficult to verify timing correctness

SOLUTION:
- Added console.log for every measure start
- Added console.log when BPM growth triggers

CODE ADDED:
```javascript
console.log(`[Loop] Global Measure ${globalMeasureRef.current}, Step ${event.stepIdx}, Beat ${event.beat}`);
console.log(`[BPM Growth] Measure ${measureCountRef.current}: ${oldBpm} → ${currentBpmRef.current} BPM`);
```

------------------------------------------------------------------------------
3. GLOBAL MEASURE COUNTER
------------------------------------------------------------------------------

ISSUE:
- event.measure was calculated locally per step
- Counter would "reset" at each phase change
- Display showed wrong measure with multi-phase flows

SOLUTION:
- Added globalMeasureRef for absolute measure tracking
- Incremented together with measureCountRef
- UI now uses globalMeasureRef.current instead of event.measure

CODE ADDED:
```typescript
const globalMeasureRef = useRef<number>(0); // Track global measure for UI display
```

CHANGES:
- Reset: globalMeasureRef.current = 0
- UI: setCurrentMeasure(globalMeasureRef.current)
- Log: "[Loop] Global Measure N"

------------------------------------------------------------------------------
4. COMPOUND TIME SIGNATURES (6/8, 9/8, 12/8)
------------------------------------------------------------------------------

ISSUE:
- 6/8 was treated as 6 beats instead of 2
- Music theory: compound time has beats grouped in triplets
- 6/8 = 2 groups of 3 eighths = 2 beats (dotted quarters)

SOLUTION:
- Added isCompoundTimeSignature() helper
- Added getEffectiveBeats() function
- Added getBeatNoteValue() function
- Added getCompoundBeatTicks() function
- Modified scheduling logic to use effective beats

CODE ADDED:
```typescript
function isCompoundTimeSignature(timeSignature: string): boolean {
  const [num] = timeSignature.split('/').map(Number);
  return num % 3 === 0 && num > 3;
}

function getEffectiveBeats(timeSignature: string): number {
  const [num] = timeSignature.split('/').map(Number);
  if (isCompoundTimeSignature(timeSignature)) {
    return num / 3; // 6/8 = 2, 9/8 = 3, 12/8 = 4
  }
  return num;
}
```

BEHAVIOR:
| Time Sig | Effective Beats | Beat Value    |
|----------|-----------------|---------------|
| 4/4      | 4               | Quarter (4n)  |
| 3/4      | 3               | Quarter (4n)  |
| 6/8      | 2               | Dotted Quarter |
| 9/8      | 3               | Dotted Quarter|
| 12/8     | 4               | Dotted Quarter|

TESTING INSTRUCTIONS
================================================================================

Follow these steps to verify the fixes:

STEP 1: START THE APP
---------------------
```bash
npm run dev
```
Open browser at http://localhost:3000

STEP 2: OPEN DEVELOPER CONSOLE
-------------------------------
Press F12 (or Cmd+Option+I on Mac)
Go to Console tab

STEP 3: TEST 1 - BPM GROWTH WITH MULTI-PHASE FLOW
---------------------------------------------------
1. Create 3 phases in Rhythmic Flow:
   - Phase 1: 4/4, 2 measures
   - Phase 2: 3/4, 1 measure
   - Phase 3: 4/4, 2 measures

2. Go to Engine tab

3. Enable BPM Growth:
   - Amount: +5
   - Every: 4
   - Unit: measures

4. Click "START SESSION"

5. Watch the Console - you should see:
   [Loop] Global Measure 1, Step 0, Beat 1
   [Loop] Global Measure 2, Step 0, Beat 1
   ...
   [Loop] Global Measure 5, Step 1, Beat 1
   [BPM Growth] Measure 5: 120 → 125 BPM

EXPECTED RESULTS:
- Counter continues 1, 2, 3, 4, 5... (does NOT reset at phase change)
- First BPM growth happens at measure 5 (after 4 measures, as configured)
- Second BPM growth happens at measure 9 (after 4 more measures)

STEP 4: TEST 2 - MEASURE COUNTER ACCURACY
------------------------------------------
1. With multi-phase flow running
2. Observe the measure counter in the UI
3. It should show: 1, 2, 3, 4, 5, 6, 7... (continuous)
4. It should NOT reset when changing between phases

STEP 5: TEST 3 - COMPOUND TIME (6/8)
-------------------------------------
1. Create a phase with 6/8 time signature
2. Listen carefully to the clicks
3. You should hear 2 clicks per measure (not 6)
4. The clicks should be evenly spaced as dotted quarters

STEP 6: TEST 4 - BPM GROWTH TIMING
-----------------------------------
1. Set BPM Growth to "Every 4 measures"
2. Start the metronome
3. First growth should happen at measure 5
4. NOT at measure 4
5. NOT immediately on start
